<task type="auto">
  <name>Task 2: Create BlogPageChrome wrapper component</name>
  <files>components/BlogPageChrome.tsx</files>
  <action>
Create `components/BlogPageChrome.tsx` as a server component (no "use client" directive) that wraps all blog and help pages with consistent page chrome:

```tsx
import { Navbar } from '@/components/Navbar'
import { Footer } from '@/components/Footer'

export function BlogPageChrome({ children }: { children: React.ReactNode }) {
  return (
    <div className="min-h-screen relative overflow-x-hidden bg-[var(--color-section-a)]">
      <Navbar />
      <main className="pt-24">{children}</main>
      <Footer />
    </div>
  )
}
```

Key points:
- No "use client" directive â€” this is a server component that imports client components (Navbar uses "use client" internally, which Next.js handles correctly at the boundary).
- `bg-[var(--color-section-a)]` sets the warm cream background from the brand palette.
- `pt-24` (96px) accounts for the fixed floating navbar height (64px nav + 24px top gap + buffer).
- `min-h-screen` ensures the footer reaches the bottom even on short-content pages.
- `overflow-x-hidden` prevents horizontal scrollbar from decorative elements.
